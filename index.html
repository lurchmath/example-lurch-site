Example Lurch Site Landing Page

What's a Lurch Site?

See this page in our documentation to learn about Lurch Sites. This page is part of an example Lurch Site to help you learn how to create your own such site.

What is this page?

Most Lurch Sites will want a landing page. This page is a simple landing page, to show you how to create one. Just place any .html file in the root folder of your Lurch Site repository. You can even copy this one and use it as a starting point. (Its code is online here.)

You can use just one file or many; it's up to you. For more information on what's possible with GitHub static site hosting, view their documentation.

What should my pages contain?

Web pages in a Lurch Site typically do three things.

  1. Describe the site, such as which course or audience it is designed for, instructor contact information, etc. How much of this you do and in what style is totally up to you.
  2. Provide a link to the copy of the Lurch app that's hosted within the site. You do so with links like these:
    • Lurch app in default (instructor) configuration
    • Lurch app in student configuration
    • Lurch app in minimal instructor configuration
    • Lurch app in minimal student configuration
  3. Provide a link to open one or more of the mathematical documents within the site, opening them in the copy of the Lurch app that's hosted within the site. We suggest putting such files in the /math folder that's already been created in this site for that purpose. You can link to such files as shown below.
    • Example link 1: Rules for propositional logic
    • Example link 2: A few uses of those rules in a proof
  4. The links in the above examples use the ?config= and ?load= query parameters. You may combine those two parameters in the same URL, such as ?config=X&load=Y.
